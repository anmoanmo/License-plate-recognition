## 3. 路线 B：如果后续出现冲突，创建统一运行环境

如果你后续升级包、换镜像源，或者出现以下情况：

- `import tensorflow, torch, cv2` 失败
- `import MainUI` 失败
- 出现明显的 ABI 或版本冲突

不要继续在现有环境里反复混装，直接新建一个统一环境。

### 建议环境名

`alpr-unified-py310`

### 创建命令

```powershell
conda create -n alpr-unified-py310 python=3.10 -y
conda activate alpr-unified-py310
python -m pip install --upgrade pip
conda install pytorch torchvision torchaudio pytorch-cuda=12.1 -c pytorch -c nvidia -y
python -m pip install tensorflow opencv-python PySide6 matplotlib scikit-learn pandas openpyxl pillow pyyaml requests scipy seaborn tqdm psutil
```

### 验证命令

```powershell
conda activate alpr-unified-py310
cd D:\ml\yolo

python -c "import cv2, tensorflow, openpyxl; print('deps ok')"
python -c "import torch, torchvision, PySide6; print('torch stack ok')"
python -c "import sys; sys.path.insert(0, r'D:\ml\yolo'); import MainUI; print('MainUI import OK')"
python MainUI.py
```

### 适用场景

- 希望一个环境里同时跑 GUI、YOLO 检测和 TensorFlow 识别
- 不想继续污染原来的 `dl-pytorch-cu121`
- 出现版本冲突后，希望直接切换到一套干净环境

## 4. 路线 C：如果只想隔离 TensorFlow，创建 TensorFlow 专用环境

## 5. 冲突判定标准

建议按下面标准判断“当前环境是否需要放弃并新建环境”：

```powershell
python -c "import tensorflow, torch, cv2; print('ok')"
python -c "import sys; sys.path.insert(0, r'D:\ml\yolo'); import MainUI; print('ok')"
```

若上述任一命令失败，或者 GUI 主入口无法正常初始化，则认为当前环境存在冲突。

一旦冲突成立：

- 不建议继续在 `dl-pytorch-cu121` 中强行卸载和回滚
- 直接切换到路线 B 或路线 C

---

## 6. TensorFlow 与 Windows 的重要说明

截至 **2026-04-06**，TensorFlow 官方安装口径可概括为：

- Windows 原生安装 `tensorflow` 时，默认按 **CPU 路线** 理解
- **Windows 原生 GPU 支持停留在 TensorFlow 2.10**
- **TensorFlow 2.11+ 若想使用 GPU，官方推荐走 WSL2**

因此，本项目在 Windows 原生环境中的现实建议是：

- `PyTorch` 继续在 Windows + CUDA 下运行
- `TensorFlow` 在 Windows 原生下按 CPU 使用
- 如果你后面非常在意 TensorFlow 的 GPU，建议单独规划 WSL2 环境，不要强行和当前项目环境混用
