## 1. 项目概述

这个项目不是单一的 YOLO 检测脚本，而是一个完整的车牌识别小系统，整体由 4 个部分串起来：

1. **YOLO 检测**：先从整张图片里找出车牌位置
2. **车牌矫正**：根据检测框进一步细化角点，并进行透视变换
3. **CNN 字符识别**：把矫正后的车牌图送入 CNN，识别 7 位字符
4. **PySide6 图形界面**：把上传图片、检测结果、车牌裁剪图和最终识别文本展示出来

一句话理解这个项目：

> 这是一个“基于 YOLO 的车牌检测 + 基于 TensorFlow/Keras 的车牌字符识别 + PySide6 GUI”的本地桌面车牌识别项目。

   - 最终识别的车牌文本

> 该项目是一个“检测模型负责定位、几何算法负责矫正、分类模型负责识别”的串联式视觉流水线。

## 6. 核心模块功能解析

## 6.1 `MainUI.py`

这是项目的主入口，也是最重要的整合层。

它主要做了几件事：

- 加载 YOLO 模型：`YOLO('best.pt')`
- 加载 CNN 模型：`keras.models.load_model('training_results/best_model.h5')`
- 绑定按钮点击事件
- 读取用户选择的图片
- 调用检测、矫正、识别流程
- 把结果显示到 Qt 标签控件上

它对应的按钮逻辑是：

- `上传图片` -> `open_src_file()`
- `车牌定位` -> `img_predict()`
- `识别车牌` -> `carNumber_predict()`
- `清空` -> `stop()`

这里有一个非常重要的使用前提：

> 正确的点击顺序必须是：**上传图片 -> 车牌定位 -> 识别车牌**

因为 `carNumber_predict()` 依赖前一步已经生成好的 `self.warped_img`。

## 6.2 `ALPR_predict.py`

这个文件负责车牌字符识别。

关键逻辑：

- 输入是矫正后的车牌图
- 每张图先被缩放到 `240x80`
- CNN 输出被 reshape 成 `7 x 65`
- 表示车牌 7 个字符位置，每个位置从 65 个类别里选 1 个
- 65 个类别包括：
  - 省份简称
  - 数字
  - 英文字母

这个函数还有一个筛选条件：

- 如果预测结果里，置信度高于等于 `0.8` 的位置少于 4 个，则该结果被丢弃

这说明它不是“有输出就认”，而是有一个简单的可信度过滤。

## 6.3 `plate_align.py`

这个文件负责把“检测框中的车牌”尽量拉正。

主要步骤：

1. 把检测框转换成初始四角点
2. 在边界框内部做自适应阈值
3. 查找最大轮廓
4. 用“点到边距离 + 点到角距离”的加权方式，优化四个角点
5. 对优化后的角点做透视变换
6. 得到标准尺寸的车牌图

这个模块是本项目里很关键的一层，因为：

- YOLO 只给出矩形框
- 真正的车牌往往有拍摄角度
- 如果不矫正，后面的字符识别准确率会下降

## 6.4 `detect.py`

这是最简单的 YOLO 检测脚本。

它只负责：

- 加载 `best.pt`
- 对一张图片做预测
- 保存预测图和裁剪结果

如果你想快速理解“YOLO 检测部分和 GUI 无关的最小闭环”，这个文件是最适合先看的。

## 6.5 `test.py`

这个文件更像是一个实验脚本，而不是正式功能模块。

它尝试了另一套基于字符轮廓的透视矫正思路，但当前存在问题：

- `fit_line_ransac()` 中直接使用了未定义的 `gray_image`

因此这个文件不能视为当前项目的可靠主流程，只能当作作者的研究/试验代码阅读。

## 6.6 `train.py`

这个文件是 YOLO 检测训练入口。

从代码上看，它计划做的是：

- 基于 `yolo11.yaml` 构建检测模型
- 用 `data.yaml` 训练车牌检测模型

但它现在不能直接跑通，原因见后文问题分析。

## 6.7 `train_cnn.py`

这是字符识别 CNN 的训练脚本。

它做的事情比较完整：

- 读取车牌图片数据集
- 根据文件名得到标签
- 做训练/验证集切分
- 构建 7 输出的 CNN 模型
- 保存最好模型和最终模型
- 导出训练日志、训练曲线、预测示例图

这里的核心思想是：

- 不是把整张车牌当成一个整体类别来分类
- 而是让网络一次输出 7 个字符位置
- 每个字符位置单独做 softmax 分类

这也是本项目最适合教学的深度学习部分之一。

## 9. 这个项目的关键是什么

如果你问“这个项目最关键的不是代码细节，而是什么”，答案是下面 5 点。

## 9.1 关键不是单个模型，而是流水线

这个项目最有价值的地方不是某一个模型本身，而是这条流程：

**检测 -> 矫正 -> 识别**

如果只做检测，没有矫正，识别效果可能差。
如果只做识别，没有检测，你根本找不到车牌区域。

## 9.2 `best.pt` 和 `best_model.h5` 是两个不同任务的核心权重

- `best.pt`：检测模型
- `training_results/best_model.h5`：字符识别模型

这两个文件缺一不可。

## 9.3 `plate_align.py` 是识别精度的重要桥梁

很多初学者会把车牌识别理解成“直接 OCR 一下”，但这个项目告诉你：

- 车牌先被正确拉正
- 再做字符识别

往往更可靠。

## 9.4 数据集组织方式决定训练代码写法

`train_cnn.py` 很依赖“文件名前 7 个字符就是标签”这个前提。

所以你在读训练代码时，要反过来理解数据组织方式，而不是只看网络结构。

## 9.5 工程路径配置和环境配置比算法更容易卡住项目

这个项目当前真正最容易让人跑不起来的，不是模型原理，而是：

- 环境缺包
- 脚本写死绝对路径
- 测试图文件名不一致
- 训练配置文件缺失

这类问题在实际工程里比算法本身更常见。

## 10. 当前仓库存在的真实问题

这里列的是我在当前仓库中实际确认到的问题，不是猜测。

### 10.1 `detect.py` 默认示例图片不存在

`detect.py` 里写的是：

- `test_images/single_blue.jpg`

但当前 `test_images/` 目录中没有这个文件。

所以它按当前代码不能直接作为成功示例使用。

### 10.2 `MainUI.py` 文件选择器看起来支持视频，但实际主流程按图片处理

界面文件选择器允许：

- `*.mp4`
- `*.mkv`
- `*.avi`
- `*.flv`
- `*.jpg`
- `*.png`

但 `img_predict()` 里实际调用的是：

- `cv2.imread(self.source)`

这说明当前主流程真正支持的是**图片**，不是视频流。

### 10.3 `train.py` 依赖仓库外绝对路径

`train.py` 里模型配置路径写成了外部绝对路径：

- `D:\2-Python\1-YOLO\YOLOv11\ultralytics-8.3.2\ultralytics\cfg\models\11\yolo11.yaml`

这意味着它不是一个可直接迁移到别人电脑上的训练脚本。

而且它还依赖：

- `data.yaml`

但仓库中当前没有找到这个文件。

所以 **YOLO 训练脚本当前不能直接复现运行**。

### 10.4 `train_cnn.py` 依赖仓库外绝对路径数据集

脚本中写的是：

- `D:\Desktop\ccpd\new_ccpd\`

但仓库本身已经有：

- `D:\ml\yolo\ccpd\new_ccpd`

也就是说：

- 仓库自带了可疑似可用的数据
- 但训练脚本却没有指向仓库内的数据目录

这会导致换机器后直接训练失败。

### 10.5 `train_cnn.py` 输出目录名与 GUI 读取目录名不一致

- `train_cnn.py` 里写的是 `training_result`
- `MainUI.py` 读取的是 `training_results/best_model.h5`

当前仓库内确实存在 `training_results/`

这说明：

- 代码和产物目录命名不统一
- 训练脚本按现状重新跑完后，可能不能直接被 GUI 无缝读取

### 10.6 `test.py` 是实验代码，不应当被当成正式模块

`test.py` 中：

- `fit_line_ransac()` 使用了未定义变量 `gray_image`

说明它目前更像实验稿，而不是稳定功能。

## 12. 最后的结论

这个项目的本质是一个面向车牌识别任务的视觉工程流程，核心价值不在于某一个神奇模型，而在于把下面 3 件事接起来了：

1. **用 YOLO 找车牌**
2. **用几何方法把车牌拉正**
3. **用 CNN 识别车牌字符**

对于学习来说，它很适合作为一个“从检测到识别的完整案例”：

- 你可以先用它学会跑项目
- 再学会读 GUI 总控代码
- 再理解检测与识别如何解耦
- 最后进入训练和工程改造

对于当前仓库来说，它已经具备“本地演示识别流程”的基础条件，但还没有达到“开箱即训、开箱即复现”的工程完备度。

如果你现在的目标是：

- **先使用项目**：直接跑 `MainUI.py`
- **先学习项目**：按“GUI -> detect.py -> plate_align.py -> ALPR_predict.py -> train_cnn.py -> train.py”顺序看
- **后续改项目**：优先修复路径、目录名和训练配置问题

这是当前最现实、最高效的路线。
